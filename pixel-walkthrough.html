Mandelbrot: one pixel worked example

This page shows one pixel from the same default view used in this project, then walks the calculation step-by-step from z0 = 0 until it escapes (|z| > 2). We use a pixel where the fixed parameter c is written as x + i y with both the real-axis value and the imaginary-axis value non-zero, and show how |z| comes from those two axes at each step. Chosen pixel, view settings, and c value

Using the same default view settings: centerX = -0.5, centerY = 0, scale = 3.2, width = height = 100 What these mean (in plain words): - centerX = -0.5: the middle of the image is centred at real value -0.5 on the complex plane. - centerY = 0: the middle of the image is centred at imaginary value 0. - scale = 3.2: the full visible width of the complex-plane window is 3.2 units. (So each pixel step changes c by about 3.2 / 100 = 0.032 in this 100-pixel view.) Pick pixel: (px, py) = (91, 51) (one row below centre so the imaginary part of c is not zero) Real axis value for c = centerX + ((px / width) - 0.5) * scale = -0.5 + ((91 / 100) - 0.5) * 3.2 = -0.5 + (0.91 - 0.5) * 3.2 = -0.5 + 1.312 = 0.812 Imaginary axis value for c = centerY + ((py / height) - 0.5) * scale * (height / width) = 0 + ((51 / 100) - 0.5) * 3.2 * 1 = 0.01 * 3.2 = 0.032 So c = 0.812 + 0.032i (0.812 on the real axis, 0.032 on the imaginary axis — the same x + i y picture as for z)

|z| from a complex number

Write any complex number as z = x + i y, where x is the value on the real axis (horizontal) and y is the value on the imaginary axis (vertical). Both x and y are ordinary real numbers. The modulus (size) of z is its distance from 0 in the plane: |z| = √(x² + y²) You square the horizontal coordinate, square the vertical coordinate, add, then take the square root. (That is not √(x² + (i y)²) — i does not go inside that sum.) The fixed parameter c for this pixel is 0.812 + 0.032i, so in the same notation: |c| = √(0.812² + 0.032²) = √0.660368 ≈ 0.8126 At each iterate zn = xn + i yn, we use the same rule: |zn| = √(xn² + yn²)

Iteration rule (written very explicitly)

General rule: zn+1 = zn2 + c Start at n = 0: z0 = 0 Now increase n by 1 each step: n = 0 gives z1 = z02 + c n = 1 gives z2 = z12 + c n = 2 gives z3 = z22 + c ...and so on. In plain words: each step squares the previous z value, then adds the same c again. Write zn = xn + i yn. Then zn2 = (xn² − yn²) + i (2 xn yn). Then zn+1 = zn2 + c adds 0.812 to the real part and 0.032 to the imaginary part (the same c every time). For this chosen pixel, c = 0.812 + 0.032i for every step.

Why the threshold is 2

For Mandelbrot calculations, |z| = 2 is the escape radius. Key fact: once any iterate has |zn| > 2, the orbit is guaranteed to run away to infinity. So in practice we can stop immediately at the first time |z| > 2. That pixel is outside the Mandelbrot set.

Step-by-step values (with |z| = √(x² + y²) each time)

z0 = 0 + 0i → x0 = 0, y0 = 0 |z0| = √(0² + 0²) = 0 z1 = z02 + c = 0 + (0.812 + 0.032i) = 0.812 + 0.032i x1 = 0.812, y1 = 0.032 |z1| = √(x1² + y1²) = √(0.812² + 0.032²) = √0.660368 ≈ 0.8126 < 2 (still bounded) z12 = (x1² − y1²) + i (2 x1 y1) = (0.659344 − 0.001024) + i (0.051968) = 0.65832 + 0.051968i z2 = z12 + c = (0.65832 + 0.812) + i (0.051968 + 0.032) = 1.47032 + 0.083968i x2 = 1.47032, y2 = 0.083968 |z2| = √(x2² + y2²) = √(1.47032² + 0.083968²) ≈ 1.4729 < 2 (still bounded) Now z22. With z2 = x2 + i y2 use (x2² − y2²) + i (2 x2 y2): x2² = 1.47032² = 2.1619417024 y2² = 0.083968² = 0.007050625024 real part of z22 = x2² − y2² = 2.1619417024 − 0.007050625024 = 2.154891077376 → about 2.154891 imaginary part of z22 = 2 x2 y2 = 2 × 1.47032 × 0.083968 = 0.24694909952 → about 0.246949 So z22 ≈ 2.154891 + 0.246949i (the 2.154891 is x2² − y2², not x2² by itself) z3 = z22 + c ≈ (2.154891 + 0.812) + i (0.246949 + 0.032) ≈ 2.966891 + 0.278949i x3 ≈ 2.966891, y3 ≈ 0.278949 |z3| = √(x3² + y3²) ≈ √(8.8024 + 0.0778) ≈ 2.980 > 2 (escape happens here) After this point we do not need more steps: crossing |z| > 2 already guarantees eventual divergence.

What this means for colour

This pixel escaped after 3 iterations, so it is outside the Mandelbrot set. With this page's palette and max iterations = 100, this gives approximately RGB(216, 116, 7) (a warm orange-gold, not black — same band as the all-real example because escape is still at step 3).

Quick contrast

If a pixel never reaches |z| > 2 within the chosen iteration limit, it is treated as interior/limit and drawn black (very dark blue in this palette).
